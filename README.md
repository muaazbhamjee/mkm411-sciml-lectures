## Overview

This repository contains the lecture notebooks and supporting code for the
Scientific Machine Learning (SciML) component of MKM411. The content spans
two 50-minute lectures delivered as interactive Jupyter notebooks, covering
artificial neural networks, physics-informed neural networks (PINNs), and
the broader SciML landscape for fluid dynamics.

These notebooks are intended for **instructor use** — they are presented live
during lectures via VS Code connected to the departmental GPU server (Mjolnir).

## Lecture Content

### Lecture 1 — Neural Networks & the SciML Landscape (50 min)

| Section | Content |
|---------|---------|
| 1 | Motivation — where classical CFD struggles and why ML is entering the space |
| 2 | ANN architecture — layers, weights, biases, forward pass |
| 3 | Activation functions — interactive explorer widget (tanh vs ReLU vs sigmoid) |
| 4 | Training — loss functions, backpropagation, Adam optimiser, interactive widgets |
| 5 | SciML landscape — PINNs, neural operators, GNNs, GANs, foundation models |

**Interactive widgets (Lecture 1):**
- **Activation function explorer** — toggle functions, show derivatives, highlight PINN requirement
- **Forward pass visualiser** — live input → layer activation bar charts
- **Loss landscape** — drag weight slider, take gradient steps, adjust learning rate

### Lecture 2 — PINNs & the Broader Field (50 min)

| Section | Content |
|---------|---------|
| 1 | From ANN to PINN — the key idea, composite loss, ANN vs PINN diagram |
| 2 | Heat equation PINN — autograd demo, chain rule, two-phase optimiser |
| 3 | Von Kármán vortex street — Raissi 2020, NS PINN, pre-trained results |
| 4 | Strengths, failure modes, and open challenges |
| 5 | Panorama — DeepONet, FNO, GNNs, GANs, Foundation Models |

## Environment Setup

### Step 1 — Clone the repository

```bash
git clone https://github.com/muaazbhamjee/mkm411-sciml-lectures.git
cd mkm411-sciml-lectures
```

### Step 2 — Use the project environment

This repository shares the same conda environment as the student project
repository (`pinn-heat-transfer`). If you have already set it up, activate it:

```bash
conda activate pinn-heat
```

If not, clone the project repository first and follow its setup instructions:

```bash
git clone https://github.com/muaazbhamjee/pinn-heat-transfer.git
cd pinn-heat-transfer
conda env create -f environment.yml
conda activate pinn-heat
pip install -r requirements.txt
cd ../mkm411-sciml-lectures
```

### Step 3 — Launch VS Code connected to Mjolnir

```bash
# On your local machine — tunnel into Mjolnir
code --remote ssh-remote+mjolnir /path/to/mkm411-sciml-lectures
```

Select the `pinn-heat` kernel in VS Code when opening the notebooks.

## Pre-lecture Setup

Complete the following **before the lecture session**, ideally the day before.

### 1 — Download the Raissi 2020 cylinder dataset

```bash
wget https://raw.githubusercontent.com/maziarraissi/PINNs/master/main/Data/cylinder_nektar_wake.mat \
     -P data/
```

Or download manually from:  
https://github.com/maziarraissi/PINNs/tree/master/main/Data

### 2 — Train the NS PINN overnight

```bash
conda activate pinn-heat
python lecture_utils/train_ns_pinn.py
```

**Estimated runtime on Mjolnir (RTX A2000):** 2–3 hours  
Output files saved to `data/`:
- `ns_pinn_pretrained.pt` — model weights
- `ns_pinn_loss_history.npz` — training loss history

> **Fallback:** If the pre-trained model is not available, Lecture 2 automatically
> displays a synthetic vortex street illustration. The lecture can proceed without
> the pre-trained model — but the pre-trained results are significantly more impressive.

### 3 — Run both notebooks end-to-end

```bash
jupyter nbconvert --to notebook --execute --inplace \
  --ExecutePreprocessor.timeout=3600 \
  Lecture_1_ANNs_and_SciML.ipynb

jupyter nbconvert --to notebook --execute --inplace \
  --ExecutePreprocessor.timeout=3600 \
  Lecture_2_PINNs_and_Beyond.ipynb
```

Confirm all cells execute cleanly and all figures render before the lecture.

## Delivery Notes

### VS Code presentation tips
- Use **Zen Mode** (`Ctrl+K Z`) for full-screen, distraction-free presentation
- Increase font size to 16–18px: `Ctrl+,` → search "font size"
- Keep a second VS Code window with pre-executed outputs open as fallback
- Run all cells top-to-bottom before the lecture to pre-populate outputs

### Live training in Lecture 1
The interactive widgets in Section 3 and 4 respond in real time — no GPU needed.
These run on CPU and respond immediately.

### Live training in Lecture 2
The autograd demo in Section 2 (MinimalHeatPINN) runs in seconds on CPU.
The NS PINN results in Section 3 are loaded from the pre-trained model —
no live training during the lecture.

### If something breaks mid-lecture
The notebooks are designed to be run top-to-bottom. If a cell fails:
1. Skip it and continue — most sections are independent
2. Fall back to the pre-executed version in a second VS Code window
3. The synthetic vortex street illustration always renders without any data files
